              # Code Review流程及规范

              # 时机

              提测前后，最晚提测后一天，自行选择时间

              # 范围

              最小范围一个commit, 或代码库中的所有代码文件

              # 方式

              审核pr， 没有备注才能合并

              # 频次

              每周一次

              # 代码风格

              代码风格「传送门」

              # 参与者

              所有前端开发人员、部分测试人员等

              # Git提交规范

              • feat(模块名称)：新功能（jira单号）
              • fix(模块名称)：修补bug（jira单号）
              • docs(模块名称)：文档（documentation）
              • style(模块名称)： 格式（不影响代码运行的变动）
              • refactor(模块名称)：重构（即不是新增功能，也不是修改bug的代码变动）
              • test(模块名称)：增加测试
              • chore：构建过程或辅助工具的变动

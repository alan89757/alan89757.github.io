浏览器缓存策略

# 为什么需要浏览器缓存

  1. 客户端(浏览器)和服务器通过http协议建立连接需要消耗时间
  2. 服务器和客户端(浏览器)数据传输也需要消耗较多时间

我们该如何减少获取资源的时长???

解决思路: 请求本地资源比远程资源更快，重用以前传输过的数据来提升访问性能

# 你会怎么设计浏览器缓存

设计逻辑: 浏览器在本地已缓存数据，直接用本地缓存数据。没有再去远程服务器获取资源。

# 是否使用浏览器缓存数据

1. cache-control: max-age=300   // HTTP 1.1版本， 缓存时长300秒
2. Expires: Wed, 01 Feb 2023 03:41:03 GMT  // HTTP 1.0版本， 缓存有效期, 具体的时间

cache-control优先级大于Expires cache-control属于http协议1.1版本的响应头 Expires属于http协议1.0版本的响应头

# 协商缓存

强制缓存过期时, 去请求服务器资源，通过对比看缓存是否过期. 未过期返回304, 告诉浏览器缓存未过期，可以直接使用浏览器缓存. 过期返回200，响应体返回服务器最新资源.

# 对比缓存是否过期

1. Etag / If-None-Match  // HTTP 1.1版本， Etag是否更新, 每次文件更新都会产生一个唯一hash值
2. Last-Modified / If-Modified-Since  // HTTP 1.0版本, 文件上次更新日期

Etag的优先级大于Last-Modified
